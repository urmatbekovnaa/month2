======>>> Первые три команды пишутся только один раз когда вы установили git на свой компьютер

git --version - показывает версию установленной программы гит
git config --global user.name "ВАШ ЛОГИН"
git config --global user.email "ВАШ EMAIL"

======>>> 1 раз при создании нового проекта

git init - инициализирует проект(.git)
git remote add origin <HTTPS link.git> - соединяет репозиторий и ваш проект
.gitignore

======>>> команды, которые пишем каждый раз когда хотим отправить изменения проекта в репозиторий:

git add file_name или git add . - готовит ваши изменения к сохранению
git commit -m 'lesson 1' - окончательно сохраняет изменения
git push origin master - отправляет сохраненные изменения (коммиты) на сервер в репозиторий

======>>> Новые команды:
git rm --cached file_name - Удаляет файл из отслеживаемой зоны и при заливке файл будет удален из репозитория
git rm --cached -r directory_name - Удаляет папку из отслеживаемой зоны и при заливке папку будет удален из репозитория
git status -s - показывает состояние индекса

======>>> Новые команды:
git clone link - клонирует репозиторий
git remote remove origin - удаляет подвязку к репозиторию
git log - показывает историю коммитов


======>>> Новые команды по веткам:
git branch - показывает все ветки
git branch name - создает новую ветку с указанным название name но не переключает нас на нее
git checkout name - переключение на ветку под названием name
git branch --delete new_features - удаляет ветку локально
git push origin --delete new_features - удаляет ветку в репозтитории
